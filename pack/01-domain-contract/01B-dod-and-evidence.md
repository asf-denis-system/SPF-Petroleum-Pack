# 01B. Переходы нефти: Definition of Done и доказательства эффекта

## Общий принцип

На каждом переходе должен быть **минимальный набор "доказательств"**, что переход реально произошёл  
(а не просто "выполнили работы").

Доказательства — это **данные/протоколы/отчёты/версии моделей**, которые фиксируют изменение метрики.

---

## T01: S0→S1 "Запланирована" → "Происследована"

### DoD (готово), если:
- ✓ Есть программа исследований/интерпретации
- ✓ Получены данные (замеры) и отчёт
- ✓ Зафиксировано, какие неопределённости уменьшили, какие остались
- ✓ На основе данных можно принимать решения о локализации/доступе

### Доказательства (что сохраняем):
- Реестр данных (что, когда, где, скважина/зонд/исследования)
- Отчёт интерпретации или протокол
- Версия модели/интерпретации "до/после" или список выводов

### Типичные БВ:
- "Нет данных по нижнему пласту" → T01 заблокирован
- "Модель слишком неопределённая" → нельзя локализовать

---

## T12: S1→S2 "Происследована" → "Локализована"

### DoD, если:
- ✓ "Знания" превращены в **локализованные запасы/зоны** с точностью, достаточной для решений
- ✓ Вычислены: объём, вероятность дренирования, скорость разработки
- ✓ Оценена точность/достоверность (доверительный интервал)
- ✓ Список решений, которые теперь можно принимать (где бурить, что воздействовать, размер скважинной сетки)

### Доказательства:
- Карта/зоны локализации (структурная/литологическая карта)
- Перечень запасов и доверия к ним (OOIP, вероятность, разброс)
- Оценка точности/достоверности
- Решения, которые стали возможны благодаря локализации

### Типичные БВ:
- "Запасы в пиках неопределённости" → нельзя инвестировать в доступ
- "Не знаем, где локализована нефть в поперечнике" → сетка скважин неоптимальна

---

## T23: S2→S3 "Локализована" → "Подготовлена к добыче"

### DoD, если:
- ✓ Подготовлен **план добычи** по объекту: вариант доступа, ограничения, CAPEX/OPEX рамки, риски
- ✓ На основе плана определены: скважинная программа, инфраструктура, темпы разработки
- ✓ Оценены: стоимость доступа, рентабельность, КИН при плане

### Доказательства:
- Объектный план (кол-во скважин, их типы, заканчивание, расход, глубины)
- Инфраструктурный план (ППД, сбор, подготовка, транспорт, пропуск)
- Режимные кривые (прогноз добычи, давления, обводнённости)
- Бюджет и риски

### Типичные БВ:
- "Доступ слишком дорог" (CAPEX высок) → нужна технология удешевления бурения
- "КИН <30%" → нужна система воздействия

---

## T34: S3→S4 "Подготовлена к добыче" → "Открыт доступ"

### DoD, если:
- ✓ Доступ реализован физически (скважины пробурены/заканчены, инфра построена)
- ✓ Готовность к эксплуатации подтверждена (испытания, комиссия)
- ✓ Контроль качества работ выполнен (критично для дальнейшей эксплуатации)

### Доказательства:
- Факт завершения строительства/заканчивания (акты, протоколы)
- Скважинная карточка (глубина, интервалы перфорации/экранов, лифт)
- Готовность инфраструктуры (опрессовки, испытания, комиссия)
- Вверка по плану (соответствие план/факт)

### Типичные БВ:
- "Скважина невостребована" (высокий MPT) → нужна интенсификация
- "Пропуск системы ограничен" → нужна расширение

---

## T45: S4→S5 "Открыт доступ" → "Продавлена/вытесняется"

### DoD, если:
- ✓ Создана управляемая система воздействия (ППД/МУН/тепловое воздействие) с начальными режимами
- ✓ Охват и приемистость ППД определены и контролируются
- ✓ Динамика фронта вытеснения мониторится (давление, обводнённость, трассеры если есть)

### Доказательства:
- Проект воздействия (скважины нагнетания, режимы, объёмы, давления)
- Данные по приемистости (кривые нагнетания, давления нагнетания, объёмы)
- Динамика фронта (карты давления, насыщения/обводнённости со временем)
- Сравнение плана vs факт по охвату

### Типичные БВ:
- "Охват ППД <50%" → нужна модель дополнительного нагнетания или расширение ППД
- "Прорыв воды в скважинах раньше прогноза" → нужна гидродинамическая коррекция модели

---

## T56: S5→S6 "Продавлена" → "Поднята на поверхность"

### DoD, если:
- ✓ Добыча устойчива и управляемая, дебиты стабильны или имеют понятный тренд
- ✓ Осложнения (непредвиденные отказы, простои) находятся в норме
- ✓ Эффект воздействия подтверждён (дебиты соответствуют прогнозам или выше)

### Доказательства:
- Дебитомер данные (суточные дебиты, стабильность)
- Обводнённость/качество продукции (откалиброванные анализы)
- Статистика простоев и причины (NPT, заклинивания, отказы оборудования)
- Сравнение фактической добычи с прогнозом из T56

### Типичные БВ:
- "Дебит упадает быстрее ожидаемого" → нужна адаптивное управление энергией
- "Высокая обводнённость рано" → нужна интенсификация или переход скважины в режим вытеснения

---

## T67: S6→S7 "Поднята" → "Поставлена на НПЗ" (товар)

### DoD, если:
- ✓ Сбор/подготовка/транспорт держат требуемый **throughput** (пропускная способность) и **качество**
- ✓ Потери в норме (испарение, спиллы, утечки <0.5%)
- ✓ Система стабильна, аварийность находится в рамках целей

### Доказательства:
- Качество подготовки (вода, соль, взвешенные вещества — в соответствии с ГОСТ)
- Потери и ограничения (где, сколько, почему)
- Пропускная способность системы vs требуемая добыча
- Статистика аварий/простоев и сбор/подготовки

### Типичные БВ:
- "Ограничение пропуска" → нужна компрессорная станция или дополнительная линия
- "Высокие потери в сборе" → нужна противокоррозионная защита или демпфирование давления

---

## Общий принцип: Доказательства — это данные, не слова

**Что не считать доказательством:**
- ✗ "Мне кажется, стало лучше"
- ✗ "По расчётам должно было…"
- ✗ Отчёт без backup'а в данных

**Что считать доказательством:**
- ✓ Выписка из SCADA/дебитомера с датами
- ✓ Протокол анализа/испытания с калибровкой
- ✓ Сравнительная таблица "до/после" с учётом запаздывания
- ✓ Версия модели/интерпретации с датой и авторством

---

## Связь с инновациями и БВ/ТВ

Любая инновация обязана указывать:
- **Какой переход T** она улучшает
- **Какие доказательства изменятся** (какие метрики, какие события в данных)
- **Кто будет владельцем доказательств** (данных, мониторинга, интерпретации)
- **Период наблюдения** (сколько времени нужно, чтобы эффект вышел из шума запаздывания)
